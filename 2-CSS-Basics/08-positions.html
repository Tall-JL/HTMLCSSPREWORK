Position

The position property specifies the type of positioning method used for an element. The four position values are:

  • fixed: this will pin something to a spot, and it will stay there as you scroll. (ex: nav bar) fixed position takes elements out of the normal document flow.
  • static:This is the defualt setting, it is not affected by top, bottom, left, and right properties. It is always positioned according to the normal flow of the page.
  • relative: does not take the element out of normal document flow. Still appears within its space/dic/container, but allows it to be offset a bit from the top, bottom, right, left. Top, right , bottom, and left properties will movve it away from its nromal position. Other content will not be adjusted to fit into any gap left by the element.
  • absolute: takes element out of the normal document flow. if a parent element position is set to relative, and the child element is set to absolute, the childs elements new (0,0) coordinate will be the upper left hand corner of the parent element instead of the while page.

The z-index property allows you to override the normal stacking order of the page. The defualt z-index is 0, and you can set that value to be positive or negative. An element with a greater z-index will stack in front of an element with an lower z-index. If two elements have the same z-index, or dont have one assigned, the element positioned last in the html will show ontop. You must have a position assigned in order to assign a z-index
